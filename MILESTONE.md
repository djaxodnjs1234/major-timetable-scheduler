- **Git 브랜치 전략**:
  - `main` — 안정 버전, 발표·시연용
  - `dev` — 통합 개발 브랜치
  - `feat/xxx`, `fix/xxx` — 개별 기능·수정 브랜치 → PR 후 `dev` 머지 → 안정화 후 `main` 머지
- **이슈 트래킹**: GitHub Issues 또는 Notion. 항목 = (제목, 담당자, 마감일, 상태).
- **백업**: 매주 금요일까지 작업분 `dev` 머지 권장. 일주일 이상 로컬에만 두지 말 것.

## 6. Definition of Done (완료 기준)

각 주차별 작업이 "완료" 되기 위한 공통 기준:

1. 코드는 `main` 또는 `dev` 브랜치에 머지됨 (개인 로컬 X)
2. 새 기능에는 README 또는 인라인 주석에 한 줄 설명 추가
3. 다른 팀원이 `git pull` 후 즉시 실행 가능 (의존성 추가 시 `requirements.txt` 또는 `*.csproj` 갱신)
4. 발표·문서 자료에는 해당 주차 진행분이 반영됨
5. 회의록에 진행 상황이 기록됨
