# 솔버 동작 이해: 규칙 2개로 보는 작은 예시

## 1. 한 줄 요약

솔버는 시간표를 먼저 다 만든 뒤 필터링하는 방식이 아닙니다.

먼저 가능한 배치 후보를 `0/1 스위치`로 만들고, 그 스위치들 사이에 규칙을 붙인 뒤, OR-Tools가 규칙을 만족하는 스위치 조합을 찾는 방식입니다.

## 2. 아주 작은 상황

예시를 작게 만들겠습니다.

과목은 2개입니다.

```text
자료구조
운영체제
```

강의실은 1개입니다.

```text
D327
```

시간은 2개입니다.

```text
월요일 3교시
월요일 4교시
```

## 3. 사용할 규칙 2개

이 예시에서는 규칙을 2개만 씁니다.

```text
규칙 1. 각 과목은 반드시 한 번 배정되어야 한다.
규칙 2. 같은 시간, 같은 강의실에는 수업이 최대 1개만 들어갈 수 있다.
```

실제 프로젝트에는 교수 중복 금지, 점심시간 금지, 블록 연속 배치 같은 규칙이 더 많지만, 원리는 이 예시와 같습니다.

## 4. 먼저 가능한 배치 후보를 스위치로 만든다

솔버는 먼저 가능한 배치 후보를 전부 스위치로 만듭니다.

```text
A = 자료구조를 월요일 3교시 D327에 넣을까?
B = 자료구조를 월요일 4교시 D327에 넣을까?

C = 운영체제를 월요일 3교시 D327에 넣을까?
D = 운영체제를 월요일 4교시 D327에 넣을까?
```

각 스위치는 0 또는 1입니다.

```text
1 = 넣는다
0 = 안 넣는다
```

아직 시간표가 만들어진 것은 아닙니다.

이 시점에는 `A`, `B`, `C`, `D`가 실제로 0인지 1인지 정해지지 않았습니다.

## 5. 규칙 1을 붙인다: 각 과목은 반드시 한 번 배정

자료구조는 월요일 3교시 또는 월요일 4교시 중 정확히 하나에 들어가야 합니다.

```text
A + B = 1
```

운영체제도 월요일 3교시 또는 월요일 4교시 중 정확히 하나에 들어가야 합니다.

```text
C + D = 1
```

여기서 `= 1`은 "정확히 하나만 선택한다"는 뜻입니다.

예를 들어 자료구조에서 가능한 상태는 다음과 같습니다.

```text
A = 1, B = 0  가능
A = 0, B = 1  가능
A = 0, B = 0  불가능
A = 1, B = 1  불가능
```

## 6. 규칙 2를 붙인다: 같은 시간, 같은 강의실 중복 금지

월요일 3교시 D327에는 자료구조와 운영체제가 동시에 들어가면 안 됩니다.

```text
A + C <= 1
```

월요일 4교시 D327에도 자료구조와 운영체제가 동시에 들어가면 안 됩니다.

```text
B + D <= 1
```

여기서 `<= 1`은 "최대 하나만 선택할 수 있다"는 뜻입니다.

## 7. 이제 OR-Tools가 값을 정해본다

OR-Tools는 `A`, `B`, `C`, `D`를 0 또는 1로 정해야 합니다.

예를 들어 OR-Tools가 먼저 이렇게 시도했다고 보겠습니다.

```text
A = 1
```

뜻은 다음과 같습니다.

```text
자료구조를 월요일 3교시 D327에 넣는다.
```

그러면 규칙 1 때문에 `B`는 0이 됩니다.

```text
A + B = 1
A = 1
따라서 B = 0
```

자료구조는 이미 월요일 3교시에 들어갔으므로, 월요일 4교시에 또 들어갈 수 없습니다.

또 규칙 2 때문에 `C`도 0이 됩니다.

```text
A + C <= 1
A = 1
따라서 C = 0
```

월요일 3교시 D327은 이미 자료구조가 쓰고 있으므로, 운영체제는 그 시간에 들어갈 수 없습니다.

그런데 운영체제는 반드시 한 번 배정되어야 합니다.

```text
C + D = 1
C = 0
따라서 D = 1
```

최종 결과는 다음과 같습니다.

```text
A = 1
B = 0
C = 0
D = 1
```

이를 시간표로 읽으면 다음과 같습니다.

```text
월요일 3교시 D327: 자료구조
월요일 4교시 D327: 운영체제
```

## 8. 불가능한 시도는 중간에 버린다

만약 OR-Tools가 이런 방향을 시도했다고 해보겠습니다.

```text
A = 1
C = 1
```

뜻은 다음과 같습니다.

```text
자료구조를 월요일 3교시 D327에 넣는다.
운영체제도 월요일 3교시 D327에 넣는다.
```

하지만 규칙 2가 있습니다.

```text
A + C <= 1
```

여기에 값을 넣으면 다음과 같습니다.

```text
1 + 1 <= 1
```

이것은 거짓입니다.

따라서 이 방향은 답이 될 수 없고, OR-Tools는 다른 방향을 찾습니다.

## 9. 중요한 오해 바로잡기

이 방식은 다음과 다릅니다.

```text
모든 시간표를 완성한다
→ 하나씩 검사한다
→ 안 되는 시간표를 버린다
```

실제 방식은 다음에 가깝습니다.

```text
가능한 배치 후보를 스위치로 만든다
→ 스위치들 사이에 규칙을 등록한다
→ OR-Tools가 스위치 값을 정하는 동안 규칙을 계속 참고한다
→ 규칙을 어기는 방향은 중간에 버린다
→ 끝까지 모순이 없으면 시간표가 된다
```

즉 `model.Add(...)`는 결과 필터가 아닙니다.

`model.Add(...)`는 OR-Tools에게 "이 규칙은 반드시 지켜라"라고 문제지에 적어두는 코드입니다.

## 10. 실제 프로젝트와 연결

실제 프로젝트에서는 스위치 이름이 `A`, `B`, `C`, `D`처럼 단순하지 않습니다.

코드에서는 주로 이런 형태를 씁니다.

```text
x[과목, 요일, 교시, 강의실]
```

예를 들면 다음과 같습니다.

```text
x[자료구조, 월요일, 3교시, D327]
```

이 값이 1이면 다음 뜻입니다.

```text
자료구조가 월요일 3교시 D327에 배정되었다.
```

이 값이 0이면 다음 뜻입니다.

```text
자료구조가 월요일 3교시 D327에 배정되지 않았다.
```

강의실 중복 금지 규칙은 코드에서 다음과 비슷하게 들어갑니다.

```python
model.Add(sum(x[(c.id, d, p, r.id)] for c in courses) <= 1)
```

이 줄은 특정 요일, 특정 교시, 특정 강의실 하나를 기준으로 모든 과목의 `x` 값을 더합니다.

예를 들어 월요일 3교시 D327이라면 다음 규칙이 만들어집니다.

```text
x[자료구조, 월요일, 3교시, D327]
+ x[운영체제, 월요일, 3교시, D327]
+ x[알고리즘, 월요일, 3교시, D327]
+ ...
<= 1
```

뜻은 다음과 같습니다.

```text
월요일 3교시 D327에는 수업이 최대 1개만 들어갈 수 있다.
```

## 11. 팀원에게 설명할 때 쓰기 좋은 문장

짧게 설명하면 이렇게 말하면 됩니다.

```text
솔버는 시간표를 전부 만든 뒤 필터링하는 게 아니다.
가능한 배치 후보를 0/1 값으로 만들고,
그 값들 사이에 규칙을 걸어둔 뒤,
OR-Tools가 규칙을 만족하는 0/1 조합을 찾는다.
켜진 값만 읽으면 최종 시간표가 된다.
```

조금 더 구현 중심으로 말하면 다음과 같습니다.

```text
우리 코드는 탐색 알고리즘 자체를 구현한 것이 아니다.
우리 코드는 OR-Tools가 풀 수 있도록 변수와 규칙을 구성한다.
실제로 조합을 찾는 일은 OR-Tools의 Solve()가 한다.
```

## 12. 용어 정리

### 스위치

어떤 배치를 선택할지 나타내는 0/1 값입니다.

이 프로젝트에서는 `x[과목, 요일, 교시, 강의실]` 같은 값이 대표적인 스위치입니다.

### 규칙

스위치들이 반드시 지켜야 하는 조건입니다.

예를 들어 "같은 시간, 같은 강의실에는 수업이 최대 1개"라는 규칙이 있습니다.

### model.Add

OR-Tools 문제지에 규칙을 추가하는 코드입니다.

결과를 필터링하는 코드가 아니라, 답을 찾기 전에 규칙을 등록하는 코드입니다.

### 해

모든 규칙을 만족하는 답입니다.

이 프로젝트에서는 완성된 시간표 하나가 해입니다.

### OR-Tools

우리가 만든 변수와 규칙을 보고 가능한 0/1 조합을 찾는 도구입니다.

OR-Tools가 시간표를 이해하는 것은 아니고, 우리가 시간표 문제를 숫자 규칙으로 바꿔서 넣어주는 것입니다.
