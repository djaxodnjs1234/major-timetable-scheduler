# 정보입력/시간표 생성 오류 식별번호

이 문서는 정보입력 화면 검증, 시간표 생성 실패 진단, 진행 로그 테스트를 위한 기준 문서다.
각 식별번호는 이후 테스트명과 사용자 안내 문구에서 그대로 참조할 수 있도록 안정적으로 유지한다.

## 식별번호 규칙

- `IE-###`: 정보입력 화면에서 잡아야 하는 오류 또는 편집 동작
- `GE-###`: 시간표 생성 전 검증 또는 생성 실패 진단에서 잡아야 하는 오류
- `GL-###`: 시간표 생성 진행사항에 표시되는 로그
- 화면 이동 경고는 현재 사용자에게 식별번호를 노출하지 않지만, 이 문서의 "화면 경고 케이스"에 함께 관리한다.
- `상태`는 `현재 처리`, `현재 일부 처리`, `추가 필요` 중 하나를 사용한다.

| IE-038 | 현재 처리 | 한 학년의 과목 최소 필요 시간칸이 전체 시간칸을 초과함 | 생성 전 검증 | 해당 학년의 필요한 최소 시간칸과 사용 가능한 시간칸을 안내하고 시수/분반/Cross 조정을 제안한다. | 2학년 등 특정 학년 과목이 40칸을 초과해 더 넣을 자리가 없는지 생성 전에 확인한다. |

위치 표기: IE 진단은 가능한 경우 `교과목 관리 > 과목`, `교수 관리 > 교수`, `강의실 관리 > 강의실`, `Cross 설정 > Cross` 형식으로 수정할 위치를 함께 표시한다. 두 과목의 고정 시간 충돌은 겹치는 요일과 교시도 표시한다.

## 테스트명 규칙

- 정보입력 테스트: `{ID}_{EnglishSummary}_{ExpectedBehavior}`
  - 예: `IE-007_BlockStructureSumMismatch_BlocksSave`
- 생성 실패 테스트: `{ID}_{EnglishSummary}_{ReportsReason}`
  - 예: `GE-021_RetakeRequiredConflict_ReportsReason`
- 진행 로그 테스트: `{ID}_{EnglishSummary}_{ShowsProgress}`
  - 예: `GL-008_Phase2ModelBuild_ShowsProgress`

## 구현 메모

- `IE-*`는 정보입력 화면 저장 검증 또는 생성 전 검증으로 연결한다.
- `GE-*`는 생성 전 검증 또는 생성 실패 진단 메시지로 연결한다.
- `GL-*`는 진행사항 로그 회귀 테스트로 연결한다.
- `현재 처리` 항목도 회귀 테스트 대상이다.
- `추가 필요` 항목은 후속 구현에서 사용자 안내 메시지와 함께 추가한다.
- 내부 상태값인 `INFEASIBLE`, `UNKNOWN`, `MODEL_INVALID`는 문서에 남기되, 실제 사용자 안내는 한국어 설명을 우선한다.
- 현재 로그의 `bound=`는 문서에는 그대로 기록한다. 후속 UI 개선 시 `허용상한=` 같은 표현으로 바꾸는 것을 권장한다.
