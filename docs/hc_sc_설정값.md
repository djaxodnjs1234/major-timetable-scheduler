# WPF 하드 조건·소프트 조건 상세 설정값

이 문서는 WPF 시간표 생성기가 실제로 사용하는 하드 조건(HC)과 소프트 조건(SC)을 설명한다.  
목적은 “조건 이름만 보는 문서”가 아니라, 입력 화면에서 어떤 값을 바꾸면 어떤 제약이 달라지는지 판단할 수 있게 하는 것이다.

## 읽는 법

- **HC(Hard Constraint)**: 반드시 지켜야 하는 조건이다. 하나라도 위반하면 시간표는 생성되지 않는다.
- **SC(Soft Constraint)**: 가능한 한 좋게 맞추는 선호 조건이다. 위반해도 해는 만들 수 있지만, 점수와 정렬 순위가 낮아진다.
- WPF 솔버는 선택된 SC를 `SC-01 -> SC-02 -> SC-03` 순서로 최적화하고, 앞 단계 결과를 잠근 뒤 다음 단계로 넘어간다.
- 번호가 비어 있는 HC는 현재 WPF 생성기에 등록되어 있지 않은 조건이다.
- 이 문서는 WPF 기준이다. Python 프로토타입의 과거 정책과 다를 수 있다.

## 공통 개념

| 항목 | 값 / 의미 |
| --- | --- |
| 요일 | `0=월`, `1=화`, `2=수`, `3=목`, `4=금` |
| 전체 교시 | `1~13` |
| 점심 | 생성 전 `고려 안 함`, `4교시`, `5교시`, `요일별 4·5교시 중 하나` 중 선택 |
| 학부 가능 교시 | `1~9` 중 선택된 점심 교시를 제외한 교시 |
| 대학원 기본 가능 교시 | `10~13` |
| 대학원 주간 overflow | 야간 수용량이 부족할 때만 점심을 제외한 `1~9`를 필요한 칸 수만큼 자동 허용 |
| 대학원 주당 시수 | 최대 `4시간` |
| 대학원 학년 값 | `5` |
| 수업 가능 교시 전체 | `1~13` 중 선택된 점심 교시를 제외한 교시 |
| 기본 교과목 ID | `ABC-01`, `ABC-02`의 기본 ID는 `ABC` |
| 강의실 수 `K` | `FixedRooms`가 비어 있으면 `1`, `FixedRooms`가 N개면 `N` |

### 시간 점유와 강의실 점유

솔버는 두 종류의 점유를 구분한다.

- `y(course, day, period)`: 과목이 그 요일·교시에 열리는지 나타내는 시간 점유다.
- `x(course, day, period, room)`: 과목이 그 요일·교시에 그 강의실을 쓰는지 나타내는 강의실 점유다.

교수 충돌, 학년 충돌, 분반 충돌, Cross, 재수강 조건은 강의실이 아니라 **시간 점유 `y`**를 본다.  
강의실 충돌, 고정 강의실, 불가 강의실은 **강의실 점유 `x`**를 본다.

### 고정 시간과 고정 강의실

- `IsFixed=true`와 `FixedSlots`는 **시간만** 고정한다.
- `FixedRooms`는 **강의실만** 지정한다.
- 고정 시간만 있고 고정 강의실이 없으면, 시간은 고정하되 방은 솔버가 자동으로 고른다.
- `FixedRooms=[R1, R2]`는 “R1 또는 R2 중 하나”가 아니라 **R1과 R2를 동시에 사용**한다.
- 과목에 `FixedRooms`가 있으면 교수의 불가 강의실(HC-21)은 적용하지 않는다. 과목 지정 강의실이 우선이다.

## HC 상세

### HC-01. 강의실 동시 사용 금지

같은 요일·교시·강의실에는 과목을 2개 이상 넣을 수 없다.

#### 적용 대상

- 모든 등록 강의실
- 모든 수업 가능 교시
- 학년이나 교수가 달라도 같은 방을 같은 시간에 쓰면 충돌이다.

#### 예시

- `월 1교시 R1`에 `자료구조`가 있으면, 같은 `월 1교시 R1`에 `운영체제`를 넣을 수 없다.
- 강의실이 `R1`, `R2` 두 개라면 서로 다른 학년 수업 두 개가 같은 시간에 각각 다른 방을 쓰는 것은 가능하다.

#### 실패 시 확인

- 고정 시간이 같은 과목 수가 그 시간에 사용할 수 있는 방 수보다 많은지 확인한다.
- 여러 과목이 같은 고정 강의실을 지정했는지 확인한다.
- 과목/교수 불가 강의실 때문에 실제 후보 방이 너무 줄었는지 확인한다.

### HC-02. 교수 동시 수업 금지

같은 교수는 같은 요일·교시에 두 수업을 맡을 수 없다.

#### 적용 대상

- 과목의 `ProfessorId`
- 과목의 `CoteachProfs`에 들어 있는 팀티칭 교수 전체

#### 예시

- `P1` 교수가 `자료구조`의 주 담당이고 `운영체제`의 팀티칭 교수라면, 두 과목은 같은 시간에 열릴 수 없다.
- 과목이 강의실 2개를 동시에 쓰더라도 교수 입장에서는 수업 1개로 계산한다.

#### 실패 시 확인

- 같은 교수의 고정 시간이 겹치는지 확인한다.
- 팀티칭 교수까지 포함해 같은 시간에 묶인 과목이 있는지 확인한다.
- 교수 ID가 잘못 들어가 서로 다른 교수인데 같은 ID로 저장되었는지 확인한다.

### HC-03. 교수 불가능 시간

교수에게 등록된 불가능 시간에는 그 교수가 맡는 과목을 배정할 수 없다.

#### 적용 대상

- 교수별 `UnavailableSlots`
- 주 담당 교수와 팀티칭 교수 모두

#### 예시

- `P1` 교수의 불가능 시간이 `수 3교시`이면, `P1`이 맡는 모든 과목은 `수 3교시`에 배정되지 않는다.
- 팀티칭 과목은 참여 교수 중 한 명이라도 불가능 시간이면 그 시간에 배정할 수 없다.

#### 실패 시 확인

- 한 교수의 불가능 시간이 너무 많아 담당 과목 시수를 채울 수 없는지 확인한다.
- 고정 시간이 교수 불가능 시간과 직접 겹치는지 확인한다.
- 팀티칭 교수의 불가능 시간까지 같이 확인한다.

### HC-04. 주당 시수 충족

각 과목은 `HoursPerWeek`만큼 정확히 배정되어야 한다.

#### 계산 방식

- 강의실 배정 수는 `HoursPerWeek × K`다.
- `K=1`이면 주당 3시간 과목은 강의실 배정 3개가 필요하다.
- `K=2`이면 주당 3시간 과목은 매 시간 방 2개를 쓰므로 강의실 배정 6개가 필요하다.

#### 예시

- `HoursPerWeek=3`, `FixedRooms=[]` -> 시간 슬롯 3개 사용
- `HoursPerWeek=3`, `FixedRooms=[R1, R2]` -> 시간 슬롯 3개, 각 슬롯에서 방 2개 동시 사용

#### 실패 시 확인

- 고정 시간 개수가 `HoursPerWeek`와 같은지 확인한다.
- 블록 구조 합계가 `HoursPerWeek`와 같은지 확인한다.
- 불가능 시간/불가 강의실 때문에 필요한 시수를 채울 공간이 남아 있는지 확인한다.

### HC-06. 블록 연속 배정과 블록 내 강의실 유지

고정 시간이 아닌 과목은 `BlockStructure`대로 연속된 교시 블록으로 배정된다.

#### 적용 대상

- `IsFixed=false`인 과목
- `BlockStructure`가 비어 있으면 전체 시수를 하나의 블록으로 본다.

#### 세부 규칙

- 블록 크기가 `2`이면 같은 요일의 연속 2교시를 사용한다.
- 블록 안에 실제 점심 교시가 끼면 사용할 수 없다. 요일별 선택 모드에서는 `3~4`와 `5~6`이 각각 해당 교시가 열린 날에만 가능하고, `4~5` 블록은 항상 불가능하다.
- 블록이 끝나는 교시가 13교시를 넘어가면 사용할 수 없다.
- 고정 강의실이 없으면 블록마다 방 1개를 고르고, 그 블록 안에서는 방을 바꾸지 않는다.
- 고정 강의실이 있으면 블록 전체가 지정된 방들을 동시에 사용한다.

#### 예시

- `HoursPerWeek=3`, `BlockStructure=[2,1]`이면 2시간 연속 블록 1개와 1시간 블록 1개가 생긴다.
- `BlockStructure=[]`, `HoursPerWeek=3`이면 3시간 연속 블록 1개로 본다.

#### 실패 시 확인

- 블록 길이가 하루의 가능 교시보다 긴지 확인한다.
- 대학원 과목은 주당 4시간까지만 허용되며, 5시간 과목은 입력/생성 전 검증에서 차단된다.
- 블록 수가 너무 많아 요일 분산(HC-20)과 함께 충돌하는지 확인한다.

### HC-08. 같은 교과목 분반 시간 중복 금지

같은 기본 교과목 ID를 가진 분반끼리는 같은 시간에 열릴 수 없다.

#### 적용 대상

- `ABC-01`, `ABC-02`처럼 기본 ID가 같은 모든 분반 쌍
- 학년이 같든 다르든 기본 ID가 같으면 적용된다.

#### 예시

- `자료구조-01`과 `자료구조-02`는 같은 시간에 열릴 수 없다.
- 같은 과목 분반은 Cross 여부와 관계없이 이 조건을 따른다.

#### 실패 시 확인

- 같은 과목 분반들의 고정 시간이 겹치는지 확인한다.
- 분반 수를 늘린 뒤 기존 고정 시간이 남아 있는지 확인한다.

### HC-11. 같은 학년 수업 시간 중복 금지

같은 학년의 서로 다른 기본 교과목은 같은 시간에 열릴 수 없다.

#### 적용 대상

- `Grade`가 같은 과목
- 기본 교과목 ID가 서로 다른 과목

#### 제외 대상

- 같은 기본 교과목 ID의 분반은 HC-08에서 처리하므로 HC-11에서는 제외한다.
- 같은 Cross 그룹에 묶인 과목은 의도적으로 같은 시간에 배치될 수 있으므로 제외한다.
- 학년이 다르면 이 조건만으로는 시간 겹침을 막지 않는다.

#### 예시

- 2학년 `자료구조`와 2학년 `운영체제`는 같은 시간에 열릴 수 없다.
- 2학년 과목과 3학년 과목은 같은 시간에 열릴 수 있다. 단, 같은 교수나 같은 방이면 다른 HC에서 막힌다.

#### 실패 시 확인

- 같은 학년 과목이 너무 많아 전체 시간 칸보다 필요한 시간이 많은지 확인한다.
- 고정 시간이 같은 학년 과목끼리 겹치는지 확인한다.
- Cross로 겹쳐도 되는 과목인데 Cross 설정이 빠졌는지 확인한다.

### HC-12. 점심시간 배정 금지

입력 화면에서 선택한 점심 교시에는 어떤 과목도 배정할 수 없다.

#### 세부 규칙

- `점심시간 고려 안 함`은 1~9교시를 모두 수업 후보로 사용한다.
- `4교시 점심`은 4교시를, `5교시 점심`은 5교시를 모든 요일에서 비운다.
- `요일별 4·5교시 중 한 교시 점심`은 솔버가 각 요일마다 정확히 한 교시를 비운다.
- 요일별 모드에서 고정 수업이 4교시를 사용하면 그날 5교시가 점심이 되고, 고정 수업이 5교시를 사용하면 그날 4교시가 점심이 된다.
- 같은 날 4교시와 5교시가 모두 고정되어 있으면 점심을 정할 수 없어 생성이 실패한다.

#### 실패 시 확인

- 고정 시간이 현재 선택한 점심 교시를 포함하는지 확인한다.
- 요일별 선택 모드라면 같은 날 4교시와 5교시가 모두 고정되었는지 확인한다.
- 블록이 4교시에서 시작해 5교시를 포함하는지 확인한다.

### HC-13. 고정 시간 배정

`IsFixed=true`인 과목은 `FixedSlots`에 입력된 모든 시간에 반드시 배정된다.

#### 중요한 점

- 이 조건은 **시간만** 고정한다.
- 강의실은 `FixedRooms`가 있으면 HC-14가, 없으면 자동 배정 조건들이 정한다.
- 다른 학년의 고정 수업끼리는 같은 시간에 있을 수 있다. 단, 교수/분반/강의실/학년 충돌은 다른 HC에서 막힌다.

#### 예시

- `FixedSlots=[월1, 월2, 수3]`이면 그 세 시간에는 반드시 해당 과목이 열린다.
- 고정 강의실이 비어 있으면 해당 시간의 방은 솔버가 가능한 방 중에서 고른다.

#### 실패 시 확인

- 고정 시간 개수가 주당 시수와 같은지 확인한다.
- 고정 시간이 블록 구조와 맞는지 확인한다.
- 고정 시간이 교수 불가능 시간, 점심시간, 학부/대학원 시간대 정책과 충돌하는지 확인한다.

### HC-14. 과목별 지정 강의실과 제외 강의실

과목에 지정 강의실이 있으면 그 방만 사용하고, 과목별 제외 강의실은 사용하지 않는다.

#### 세부 규칙

- `FixedRooms`가 N개면 그 N개 방을 매 수업 시간에 동시에 사용한다.
- `UnavailableRooms`에 들어간 방은 그 과목이 사용할 수 없다.
- 과목의 `FixedRooms`와 `UnavailableRooms`가 겹치면 해가 없다.
- 과목의 `FixedRooms`가 있으면 교수의 불가 강의실보다 과목 지정 강의실이 우선한다.

#### 예시

- `FixedRooms=[R1]`이면 해당 과목은 항상 `R1`만 사용한다.
- `FixedRooms=[R1, R2]`이면 해당 과목은 매 시간 `R1`과 `R2`를 동시에 사용한다.
- `UnavailableRooms=[R3]`이면 자동 배정 때 `R3`은 후보에서 빠진다.

#### 실패 시 확인

- 지정 강의실이 실제 강의실 목록에 있는지 확인한다.
- 지정 강의실과 제외 강의실이 겹치는지 확인한다.
- 같은 시간에 같은 지정 강의실을 쓰는 고정 과목이 있는지 확인한다.

### HC-15. 같은 교수의 2개 분반 인접 배정

같은 과목의 2개 분반이 같은 교수진과 같은 블록 구조를 가지면, 같은 날 앞뒤로 붙여 배정한다.

#### 적용 조건

- 같은 기본 교과목 ID의 분반이 정확히 2개다.
- 두 분반의 주 담당 교수와 팀티칭 교수 집합이 완전히 같다.
- 두 분반의 블록 구조가 완전히 같다.
- 두 분반 모두 고정 시간이 아니다.
- 블록 크기 3은 제외한다.

#### 예시

- `자료구조-01`, `자료구조-02`가 둘 다 `P1` 교수이고 블록 구조가 `[2,1]`이면, 2시간 블록끼리 같은 날 앞뒤로 붙고 1시간 블록도 앞뒤로 붙는다.
- 블록 크기 2에서 한 분반이 `월 1~2교시`면 다른 분반은 가능한 경우 `월 3~4교시`에 붙는다.

#### 실패 시 확인

- 교수 불가능 시간이 많아 인접 가능한 시간이 사라졌는지 확인한다.
- 한 분반만 고정 시간이 켜져 있어 조건이 적용되지 않는지 확인한다.
- 분반이 3개 이상이면 이 조건이 적용되지 않는다.

### HC-16. Cross 교과목 순환 동시간 배정

Cross로 묶인 두 교과목은 분반을 한 칸 밀어서 같은 시간에 배정한다.

#### WPF 입력 기준

- WPF 화면에서는 Cross 과목을 정확히 2개 선택한다.
- 두 과목은 분반 수가 같아야 한다.
- 두 과목은 총 시수가 같아야 한다.
- 두 과목은 블록 구조가 같아야 한다.
- 이미 다른 Cross에 들어간 과목은 다시 선택할 수 없다.

#### 매칭 방식

분반이 2개인 경우:

- A-01은 B-02와 같은 시간
- A-02는 B-01과 같은 시간

분반이 3개인 경우:

- A-01은 B-02와 같은 시간
- A-02는 B-03과 같은 시간
- A-03은 B-01과 같은 시간

#### 왜 필요한가

같은 학년 과목이라도 Cross로 묶으면 HC-11의 같은 학년 중복 금지에서 제외된다. 대신 Cross 규칙이 의도한 조합대로 같은 시간에 배치한다.

#### 실패 시 확인

- Cross 대상 과목의 분반 수, 시수, 블록 구조가 같은지 확인한다.
- Cross 과목들이 같은 고정 강의실을 공유하는지 확인한다.
- 시간고정이 Cross의 순환 매칭과 맞지 않는지 확인한다.

### HC-17. 재수강생을 위한 안전 분반 확보

재수강 대상 과목의 분반 중 최소 1개는 현재 학년 전공 과목들과 시간이 겹치지 않아야 한다.

#### 적용 조건

- 생성 옵션에서 `재수강생 고려`가 켜져 있어야 한다.
- 대상은 전필 과목이다.
- 낮은 학년 전필은 그보다 높은 학년 학생이 재수강할 수 있다고 본다.
- 별도 `RetakeScenario`가 있으면 자동 조건과 합쳐서 적용한다.

#### 자동 생성 예시

- 1학년 전필 A -> 2학년, 3학년, 4학년 학생의 재수강 가능성을 고려한다.
- 2학년 전필 B -> 3학년, 4학년 학생의 재수강 가능성을 고려한다.
- 4학년 전필은 더 높은 학부 학년이 없으므로 자동 재수강 대상이 생기지 않는다.

#### 안전 분반의 의미

- 모든 분반이 안전해야 하는 것은 아니다.
- 재수강생이 선택할 수 있는 분반이 **최소 1개** 있으면 된다.
- 안전 분반은 해당 학년의 다른 전필들과 겹치는 요일·교시가 `0개`여야 한다.

#### 예시

- 3학년 학생이 2학년 전필 `자료구조`를 재수강할 수 있어야 한다.
- `자료구조`가 2분반이면, 둘 중 최소 1분반은 3학년 전필들과 시간이 겹치지 않아야 한다.

#### 실패 시 확인

- 재수강생 고려 옵션이 꼭 필요한지 확인한다.
- 현재 학년 전필들의 고정 시간이 너무 빡빡한지 확인한다.
- 재수강 대상 과목의 분반을 늘릴 수 있는지 확인한다.
- 전필로 잘못 지정된 과목이 있는지 확인한다.

### HC-19. 2시간 블록 시작 교시와 Cross 1시간 블록 정렬

2시간 블록은 정해진 시작 교시에서만 시작하고, Cross에 포함된 1시간 블록은 서로 이웃한 시간에 배정한다.

#### 2시간 블록 시작 교시

허용 시작 교시는 선택한 점심 정책에서 연속된 두 교시를 기준으로 계산한다.

- 점심 미적용 학부: `1~2`, `3~4`, `5~6`, `7~8`
- 4교시 점심 학부: `1~2`, `5~6`, `7~8`
- 5교시 점심 학부: `1~2`, `3~4`, `6~7`, `8~9`
- 요일별 4·5교시 모드는 그날 실제로 열린 연속 구간만 사용하며 `4~5`는 항상 불가능하다.
- 대학원 예: `10~11`, `12~13`

#### Cross 1시간 블록 정렬

Cross 멤버 과목이 2분반이고 블록 구조가 같을 때, 1시간 블록은 서로 이웃하게 배치한다.

- 첫 분반이 `{1,3,6,8,10,12}`에서 시작하면 둘째 분반은 같은 날 `+1`교시
- 첫 분반이 `{2,4,7,9,11,13}`에서 시작하면 둘째 분반은 같은 날 `-1`교시

#### 실패 시 확인

- 2시간 블록이 허용 시작 교시 외에 고정되어 있는지 확인한다.
- Cross 과목의 1시간 블록이 서로 이웃할 수 있는 시간이 남아 있는지 확인한다.

### HC-20. 같은 과목의 블록 요일 분산

블록이 2개 이상인 자동 배정 과목은 블록들을 서로 다른 요일에 배정한다.

#### 적용 대상

- `IsFixed=false`
- 블록 수가 2개 이상인 과목

#### 예시

- `BlockStructure=[2,1]`이면 2시간 블록과 1시간 블록은 서로 다른 요일에 배치된다.
- `BlockStructure=[1,1,1]`이면 세 블록은 모두 다른 요일에 배치된다.

#### 실패 시 확인

- 블록 수가 5개를 넘으면 5일 안에 모두 다른 요일로 배치할 수 없다.
- 고정 시간이 켜진 과목은 이 조건으로 요일을 조정하지 않는다.

### HC-21. 교수별 불가 강의실

자동 강의실 배정 과목은 주 담당 교수의 불가 강의실을 피한다.

#### 적용 대상

- `FixedRooms`가 비어 있는 과목
- 주 담당 교수 `ProfessorId`의 `UnavailableRooms`

#### 제외 대상

- 과목에 `FixedRooms`가 있으면 이 조건은 적용하지 않는다.
- 팀티칭 교수의 불가 강의실은 이 조건에서 참조하지 않는다.

#### 예시

- `P1` 교수의 불가 강의실이 `R3`이면, `P1`이 주 담당인 자동 배정 과목은 `R3`을 쓰지 않는다.
- 과목이 `FixedRooms=[R3]`이면 과목 지정 방이 우선하므로 HC-21은 적용되지 않는다.

#### 실패 시 확인

- 교수 불가 강의실과 과목 불가 강의실을 합쳤을 때 후보 방이 하나도 남지 않는지 확인한다.
- 고정 강의실을 써야 하는 과목인지, 자동 배정이어야 하는 과목인지 확인한다.

### HC-22. 자동 배정 과목의 공통 강의실

자동 강의실 배정 과목은 하나의 공통 강의실을 사용한다. 한 과목 안의 여러 블록도 같은 방을 쓰고, 같은 교과목의 자동 배정 분반들도 같은 방을 쓴다.

#### 적용 대상

- `FixedRooms`가 비어 있는 자동 배정 과목
- 같은 기본 교과목 ID의 분반이 여러 개면 해당 분반들 중 어느 분반도 `FixedRooms`를 갖지 않음

#### 세부 규칙

- 과목 그룹마다 공통 강의실 1개를 고른다.
- 각 과목과 분반은 모든 배정 시간에 그 공통 강의실만 사용한다.
- 과목/교수 불가 강의실 때문에 공통 강의실 후보가 없어지면 생성이 실패할 수 있다.

#### 예시

- `정보보안과블록체인`이 `BlockStructure=[1,2]`인 자동 배정 과목이면, 1시간 블록과 2시간 블록이 같은 강의실을 쓴다.
- `자료구조-01`, `자료구조-02`가 자동 배정이면 둘 다 같은 강의실을 쓰도록 묶인다.
- `자료구조-01`만 고정 강의실이 있더라도 해당 과목 그룹 전체에는 HC-22가 적용되지 않는다.

#### 실패 시 확인

- 한 과목의 여러 블록이 모두 들어갈 수 있는 공통 강의실 후보가 남아 있는지 확인한다.
- 분반별 과목 불가 강의실이 서로 달라 공통 후보가 없어졌는지 확인한다.
- 주 담당 교수의 불가 강의실 때문에 공통 후보가 사라졌는지 확인한다.

### HC-23. 학부·대학원 수업 시간대 분리

학부 과목은 주간에만 배정한다. 대학원 과목은 야간을 우선 사용하되, 대학원 야간 수용량이 부족할 때만 필요한 수만큼 주간 배정을 자동 허용한다.

#### 세부 규칙

- `Grade=5`이면 대학원 과목으로 본다.
- 대학원 과목은 기본적으로 `10~13교시`를 사용한다.
- 대학원 과목의 최소 필요 시간칸이 야간 수용량을 넘거나, 3시간 연속 블록이 야간에 모두 들어갈 수 없으면 주간 overflow가 켜진다.
- 주간 overflow가 켜져도 솔버는 계산된 overflow 칸 수까지만 대학원 주간 배정을 허용한다.
- 대학원 과목은 주당 `4시간`까지만 설정할 수 있고, `5시간` 이상은 입력/생성 전 오류로 처리한다.
- `Grade=5`가 아닌 과목은 학부/일반 과목으로 보고 `1~9교시` 중 실제 점심 교시를 제외해 사용할 수 있다.
- 고정 점심 모드에서는 모든 요일의 4교시 또는 5교시가 비고, 요일별 모드에서는 각 요일마다 둘 중 하나만 빈다.

#### 예시

- 대학원 과목 수가 야간 20칸 안에 들어가면 대학원 수업은 `10~13교시`에만 배정된다.
- 대학원 필요 시간칸이 야간 20칸을 초과하면 초과한 칸만 점심을 제외한 `1~9교시`에 배정될 수 있다.
- 야간 overflow가 필요하지 않은 상태에서 대학원 과목을 `월 3교시`에 고정하면 생성이 실패한다.
- 학부 과목을 `월 10교시`에 고정하면 생성이 실패한다.

#### 실패 시 확인

- 학년 값이 의도와 맞는지 확인한다.
- 대학원 과목의 주당 시수가 5시간 이상인지 확인한다.
- 주간 overflow가 켜진 상태에서도 교수 불가능 시간, Cross, 고정 시간 때문에 배치가 막히는지 확인한다.

## SC 상세

### SC-01. 월요일 오전·금요일 오후 회피

월요일 오전과 금요일 오후에 수업이 몰리는 것을 피한다.

#### 불리한 시간

- 월요일 `1~4교시`
- 금요일 `6~9교시`

#### 솔버 계산

- 불리한 시간의 강의실 배정 1개마다 패널티 1점이다.
- 다중 강의실 과목은 방 수만큼 패널티가 늘어난다.
- 최적값보다 추가 위반을 허용하지 않는다. (`Sc01SlackAbs=0`)

#### 화면 점수

- 점수 범위는 `0.0~1.0`이다.
- `1.0`에 가까울수록 월 오전·금 오후를 잘 피한 것이다.
- 화면 점수는 `max(0, 1 - 불리한 배정 수 / 20)`이다.

#### 사용자가 조정할 수 있는 것

- 해당 시간대에 고정된 과목을 줄인다.
- 교수 불가능 시간이 너무 많아 다른 시간 선택지가 없는지 확인한다.

### SC-02. 교수 수업 요일 수 3일 이하 선호

교수의 수업 요일 수가 3일을 넘지 않도록 선호한다.

#### 적용 대상

- 고정 시간이 아닌 과목만 계산한다.
- 주 담당 교수만 계산한다.
- 팀티칭 추가 교수는 계산에서 제외한다.
- 자동 배정 과목을 2개 이상 맡은 교수만 대상이다.

#### 솔버 계산

- 기준 요일 수는 `3일`이다.
- 교수 1명의 패널티는 `max(0, 사용 요일 수 - 3)`이다.
- 5일제이므로 교수 1명당 최대 초과값은 2일이다.
- 최적값보다 추가 초과 1일을 허용한다. (`Sc02SlackAbs=1`)

#### 화면 점수

- `3일 이하`면 해당 교수는 감점이 없다.
- `4일`이면 초과 1일, `5일`이면 초과 2일이다.
- 점수는 대상 교수들의 총 초과 일수를 기준으로 계산한다.

#### 사용자가 조정할 수 있는 것

- 교수 불가능 시간을 줄여 같은 요일 안에 배치할 여지를 만든다.
- 같은 교수의 과목이 너무 많으면 고정 시간을 줄이거나 담당 교수를 나눈다.

### SC-03. 같은 과목 블록의 요일 간격 2일 선호

블록이 여러 개인 과목은 블록들이 너무 붙거나 너무 멀지 않게, 요일 차이 2일을 선호한다.

#### 적용 대상

- `BlockStructure`에 블록이 2개 이상인 과목
- 솔버 패널티는 자동 배정 과목 중 블록 요일 변수가 있는 과목에 적용된다.
- 화면 점수 계산은 배정 결과를 기준으로 블록이 여러 개인 과목을 평가한다.

#### 선호 점수

| 요일 차이 | 화면 점수 해석 |
| --- | --- |
| 2일 | 가장 좋음 (`1.0`) |
| 1일 | 꽤 좋음 (`0.8`) |
| 3~4일 | 낮음 (`0.2`) |
| 그 외 | 나쁨 (`0.0`) |

#### 솔버 계산

- 솔버는 요일 차이 2일을 최우선으로 선호한다.
- 요일 차이 1일은 작은 패널티다.
- 요일 차이 0일, 3일 이상은 큰 패널티다.
- 최적값보다 추가 위반을 허용하지 않는다. (`Sc03SlackAbs=0`)

#### 사용자가 조정할 수 있는 것

- 블록 구조가 너무 세분화되어 있으면 줄인다.
- 교수 불가능 시간이나 고정 시간이 많아 특정 요일에만 몰리는지 확인한다.

## 종합 점수와 정렬

| 항목 | 값 |
| --- | --- |
| 개별 SC 점수 범위 | 각각 `0.0~1.0` |
| SC 가중치 | SC-01, SC-02, SC-03 모두 `1` |
| SC 지수 | SC-01, SC-02, SC-03 모두 `1` |
| 종합 최고점 | `3.0` |
| 종합 점수 | `SC-01 + SC-02 + SC-03` |
| 백분율 표시 | `종합 점수 / 3.0 × 100` |

SC는 “해가 있는 후보들 사이의 선호”다.  
HC 때문에 해가 없으면 SC 점수는 계산할 대상이 없다. 먼저 HC 충돌을 줄이고, 그 다음 SC 점수를 조정하는 순서로 보는 것이 좋다.

## 자주 헷갈리는 케이스

### 다른 학년 고정 수업이 같은 시간이어도 되는가?

가능하다. HC-11은 같은 학년만 막는다.  
다만 같은 교수(HC-02), 같은 방(HC-01), 같은 과목 분반(HC-08), 같은 고정 강의실(HC-14)이면 다른 학년이어도 막힌다.

### 고정 강의실 여러 개는 후보 목록인가?

아니다. `FixedRooms=[R1, R2]`는 매 수업 시간에 `R1`과 `R2`를 동시에 쓰겠다는 뜻이다.

### 교수별 허용 강의실 기능이 있는가?

없다. 현재 WPF에는 교수별 허용 강의실이 없다.  
남아 있는 것은 교수별 **불가 강의실**이며, 이것도 자동 강의실 배정 과목에만 적용된다.

### 재수강생 고려를 켜면 모든 재수강 과목이 안 겹치나?

아니다. 재수강 대상 과목의 모든 분반을 안전하게 만드는 것이 아니라, 최소 1개 안전 분반을 확보한다.

### Cross는 같은 학년 중복 금지를 무시하는가?

Cross로 묶인 과목끼리는 HC-11 비교에서 제외된다.  
대신 HC-16이 정한 순환 매칭대로 같은 시간에 배치된다.
