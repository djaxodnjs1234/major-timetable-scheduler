# WPF 하드 조건·소프트 조건 상세 설정값

이 문서는 현재 WPF 솔버에 등록된 조건과 수치만 정리한다. 하드 조건(HC)은 하나라도 위반하면 해가 될 수 없고, 소프트 조건(SC)은 선택 시 더 좋은 시간표를 고르는 기준이다.

## 조건별 상세 내용

### 공통 시간 기준

| 항목 | 값 |
| --- | --- |
| 요일 | `0~4` = 월~금, 총 5일 |
| 전체 교시 | `1~13`, 총 13교시 |
| 점심 | 5교시 (`13:00~14:00`) |
| 수업 가능 교시 | `1, 2, 3, 4, 6, 7, 8, 9, 10, 11, 12, 13`, 총 12교시 |
| 학부 수업 가능 교시 | `1~4, 6~9`, 총 8교시 (`09:00~13:00`, `14:00~18:00`) |
| 대학원 수업 가능 교시 | `10~13`, 총 4교시 (`18:00~22:00`) |
| 강의실 수 `K` | 과목 `FixedRooms` 수. 비어 있으면 `K=1`, 방이 2개면 `K=2` |

같은 과목의 같은 시간에 배정되는 방 수는 항상 `K`개다. 따라서 `K=2`인 과목은 한 시간에 강의실 2개를 동시에 사용한다.

### HC-01. 강의실 동시 사용 금지

같은 요일·교시·강의실에는 서로 다른 과목을 2개 이상 배정할 수 없다.

#### 상세 설정값·수치

- 적용 범위: 5일 × 수업 가능 12교시 × 등록 강의실 수.
- 각 요일·교시·강의실 조합의 과목 수는 `1개 이하`다.
- 같은 과목의 다중 강의실 배정은 서로 다른 강의실을 쓰므로 허용된다.

### HC-02. 교수 동시 수업 금지

주 담당 교수와 팀티칭 교수를 포함해, 같은 교수가 같은 요일·교시에 두 과목을 맡을 수 없다.

#### 상세 설정값·수치

- 적용 교수: `ProfessorId` 1명과 `CoteachProfs`의 모든 교수.
- 각 교수의 같은 요일·교시 수업 수는 `1개 이하`다.
- 다중 강의실 과목은 방 수와 관계없이 하나의 수업으로 계산한다.

### HC-03. 교수 불가능 시간

교수에게 등록된 불가능 시간에는 해당 교수가 맡는 과목을 배정할 수 없다.

#### 상세 설정값·수치

- 입력값: 교수별 `UnavailableSlots`의 `(요일 0~4, 교시 1~13)` 목록.
- 목록의 슬롯마다 해당 과목의 모든 강의실 배정값은 `0`이다.
- 주 담당 교수와 팀티칭 교수 모두에 적용된다.
- 5교시는 HC-12에서 별도로 전면 금지한다.

### HC-04. 주당 시수 충족

각 과목은 설정한 주당 시수를 정확히 채워야 한다.

#### 상세 설정값·수치

- 공식: 전체 강의실 배정 수 = `HoursPerWeek × K`.
- 예: 주당 3시간, `K=1`이면 배정 3개다.
- 예: 주당 3시간, 고정 강의실 2개로 `K=2`이면 배정 6개이며, 실제 수업 시간은 3개 슬롯이다.

### HC-06. 블록 연속 배정과 블록 내 강의실 유지

고정 시간이 아닌 과목은 `BlockStructure`대로 연속 교시 블록으로 배정한다. 블록 하나는 한 요일에 시작하며, 자동 배정 강의실은 블록이 끝날 때까지 바뀌지 않는다.

#### 상세 설정값·수치

- `BlockStructure`가 비어 있으면 블록은 `HoursPerWeek` 크기 1개다.
- 블록 크기가 `b`이면 같은 요일의 연속 `b`교시를 사용한다.
- 블록 안에 5교시가 포함되거나 13교시를 넘는 시작점은 사용할 수 없다.
- 각 블록의 시작점은 정확히 `1개`다.
- `FixedRooms`가 있으면 블록의 모든 교시에 지정 방 `K`개를 동시에 사용한다.
- `FixedRooms`가 없으면 블록마다 강의실 `1개`를 골라 모든 교시에서 유지한다.
- `IsFixed=true` 과목은 이 조건의 블록 배정 대상이 아니다.

### HC-08. 같은 교과목 분반 시간 중복 금지

같은 기본 교과목 ID를 가진 분반끼리는 같은 요일·교시에 열 수 없다.

#### 상세 설정값·수치

- 기본 교과목 ID는 마지막 `-` 앞부분이다. 예: `ABC-01`, `ABC-02`의 기본 ID는 모두 `ABC`다.
- 같은 기본 ID의 분반 2개가 같은 요일·교시에 함께 점유할 수 있는 수는 `0개`다.
- 분반 수가 2개 이상이면 모든 분반 쌍에 적용된다.

### HC-11. 같은 학년 수업 시간 중복 금지

같은 학년의 서로 다른 기본 교과목은 같은 요일·교시에 열 수 없다. 단, 같은 교과목 분반과 Cross 그룹으로 묶인 교과목은 이 조건의 비교 대상에서 제외한다.

#### 상세 설정값·수치

- 비교 학년: 과목 `Grade` 값이 같은 과목끼리.
- 각 비교 과목 쌍의 같은 요일·교시 동시 점유 수는 `0개`다.
- 기본 교과목 ID가 같으면 HC-08이 적용되므로 HC-11에서는 제외한다.
- 두 기본 ID가 같은 Cross 그룹 `1개`에 함께 있으면 HC-11에서는 제외한다.

### HC-12. 점심시간 배정 금지

점심시간에는 어떤 과목도 배정할 수 없다.

#### 상세 설정값·수치

- 점심은 5교시다.
- 시간 범위는 `13:00~14:00`이다.
- 5일 × 모든 강의실에서 과목 배정 수는 `0개`다.

### HC-13. 고정 시간 배정

`IsFixed=true`인 과목은 입력된 모든 고정 시간 슬롯을 반드시 점유한다.

#### 상세 설정값·수치

- 입력값: 과목별 `FixedSlots`의 `(요일, 교시)` 목록.
- 목록의 각 슬롯은 점유값 `1`이어야 한다.
- 강의실은 이 조건에서 정하지 않는다. 지정 방은 HC-14, 자동 방은 HC-21과 HC-22가 정한다.
- 고정 슬롯이 5교시이면 HC-12와 충돌하므로 해가 없다.

### HC-14. 과목별 지정 강의실과 제외 강의실

과목에 지정 강의실이 있으면 그 방만 사용하고, 과목별 제외 강의실은 사용하지 않는다.

#### 상세 설정값·수치

- `FixedRooms`가 `N`개면 그 외 강의실 배정값은 모두 `0`이고, `K=N`이다.
- 예: `FixedRooms=[R1, R2]`이면 실제 수업 슬롯마다 `R1`과 `R2`를 동시에 사용한다.
- `UnavailableRooms`에 등록된 강의실의 모든 요일·수업 가능 교시 배정값은 `0`이다.
- 존재하지 않는 방 ID가 `UnavailableRooms`에 있어도 등록 강의실 목록에 없으므로 영향이 없다.

### HC-15. 같은 교수의 2개 분반 인접 배정

동일 교과목의 2개 분반이 같은 교수진과 같은 블록 구조를 가지면, 3시간 블록을 제외한 각 블록을 같은 날 앞뒤로 붙여 배정한다.

#### 상세 설정값·수치

- 적용 분반 수: 정확히 `2개`.
- 두 분반의 주 담당·팀티칭 교수 집합이 완전히 같고, 적어도 교수 `1명`이 있어야 한다.
- 두 분반의 블록 구조도 완전히 같아야 한다.
- 블록 크기가 `b`이면 두 블록의 시작 교시 차이는 같은 날 `+b` 또는 `-b`다.
- 블록 크기 `3`은 이 조건에서 제외한다.
- 두 분반 중 하나라도 `IsFixed=true`이면 적용하지 않는다.

### HC-16. Cross 교과목 순환 동시간 배정

Cross 그룹의 교과목들은 분반 순서를 한 칸 순환시켜 같은 시간에 배정한다.

#### 상세 설정값·수치

- Cross 그룹은 서로 다른 기본 교과목 ID를 최소 `2개` 포함해야 한다.
- 빈 ID, 중복 ID, 존재하지 않는 기본 ID가 있으면 모델 생성 단계에서 오류가 난다.
- 각 교과목의 분반은 `Section` 오름차순으로 정렬한다.
- `K`는 그룹에 속한 교과목들 중 가장 적은 분반 수다.
- 인접한 두 교과목 그룹에서 앞 교과목의 `k`번째 분반은 뒤 교과목의 `(k+1) mod K`번째 분반과 같은 요일·교시를 점유한다.

### HC-17. 재수강생을 위한 안전 분반 확보

재수강 대상 교과목의 분반 중 적어도 하나는 해당 학년 전공 과목과 시간이 겹치지 않아야 한다.

#### 상세 설정값·수치

- 입력값: `RetakeScenario(CurrentGrade, RetakeBaseId)`.
- 대상 전공: `CurrentGrade`가 같은 전공 과목 중 `RetakeBaseId`와 기본 ID가 다른 과목.
- 재수강 대상의 모든 분반 중 전공 과목과 겹치는 요일·교시 수가 `0`인 분반이 `1개 이상` 있어야 한다.
- 재수강생 고려 옵션이 켜진 경우, 자동 시나리오는 학부 `1~4학년`의 전공 과목마다 상위 학년에 대해 생성된다.
- 재수강생 고려 옵션이 꺼져 있으면 솔버에는 이 조건이 전달되지 않는다.

### HC-19. 2시간 블록 시작 교시와 Cross 1시간 블록 정렬

2시간 블록은 정해진 시작 교시에서만 시작한다. Cross에 포함된 2개 분반의 1시간 블록은 서로 이웃한 교시에 배정한다.

#### 상세 설정값·수치

- 2시간 블록 시작 교시 허용값: `{1, 3, 6, 8, 10, 12}`, 총 6개.
- 위 값이 아닌 시작 교시는 2시간 블록에 사용할 수 없다.
- Cross 멤버인 기본 교과목의 분반이 정확히 2개이고 블록 구조가 같을 때만 1시간 블록 정렬을 적용한다.
- 첫 분반의 1시간 블록 시작이 `{1, 3, 6, 8, 10, 12}`이면 둘째 분반은 같은 날 `+1`교시다.
- 첫 분반의 시작이 `{2, 4, 7, 9, 11, 13}`이면 둘째 분반은 같은 날 `-1`교시다.

### HC-20. 같은 과목의 블록 요일 분산

블록이 2개 이상인 과목은 블록들을 서로 다른 요일에 배정한다.

#### 상세 설정값·수치

- 블록 수가 `2개 이상`일 때 적용한다.
- 모든 블록의 요일 값은 서로 달라야 한다.
- 사용 가능한 요일은 5일이므로 블록이 `6개 이상`이면 이 조건만으로 해가 없다.
- `IsFixed=true` 과목은 HC-06에서 블록 요일 변수를 만들지 않으므로 이 조건의 적용 대상이 아니다.

### HC-21. 교수별 사용 가능·불가능 강의실

자동 강의실 배정 과목은 주 담당 교수의 사용 가능 강의실과 사용 불가능 강의실을 따른다.

#### 상세 설정값·수치

- 적용 대상: `FixedRooms`가 비어 있는 과목만.
- 참조 교수: 주 담당 `ProfessorId` 1명만. 팀티칭 교수의 강의실 목록은 참조하지 않는다.
- `AllowedRooms`가 비어 있으면 모든 등록 강의실이 후보가 된다.
- `AllowedRooms`가 `N`개이면 그 `N`개 강의실만 후보가 된다.
- `UnavailableRooms`에 든 강의실은 모든 요일·수업 가능 교시에서 배정값 `0`이다.
- 허용 목록과 불가능 목록에 같은 방이 있으면 불가능 목록이 우선한다.

### HC-22. 같은 교과목 분반의 공통 강의실

자동 강의실 배정인 같은 교과목 분반들은 하나의 공통 강의실을 사용한다.

#### 상세 설정값·수치

- 적용 분반 수: `2개 이상`.
- 같은 기본 교과목 ID의 모든 분반에 `FixedRooms`가 비어 있어야 한다.
- 모든 등록 강의실 중 공통 강의실은 정확히 `1개` 선택된다.
- 각 분반은 배정된 모든 요일·교시에서 선택된 공통 강의실만 사용할 수 있다.
- 분반 중 하나라도 `FixedRooms`가 있으면 이 조건은 해당 교과목 전체에 적용하지 않는다.

### HC-23. 학부·대학원 수업 시간대 분리

대학원 과목은 야간에만, 대학원 이외 과목은 주간에만 배정한다.

#### 상세 설정값·수치

- 대학원 학년 값은 `5`다.
- 대학원 허용 교시: `10~13`, 총 4교시 (`18:00~22:00`).
- 대학원 이외 과목 허용 교시: `1~4, 6~9`, 총 8교시 (`09:00~13:00`, `14:00~18:00`).
- 5교시는 학부·대학원 모두 HC-12에 의해 사용할 수 없다.

### SC-01. 월요일 오전·금요일 오후 회피

월요일 오전과 금요일 오후에 배정된 수업 수를 줄인다.

#### 상세 설정값·수치

- 불리한 슬롯: 월요일(`day=0`) 1~4교시 4개, 금요일(`day=4`) 6~9교시 4개.
- 전체 불리한 슬롯 수는 `8개`다.
- 솔버 패널티: 불리한 슬롯에 배정된 강의실 배정 `1개`당 `1점`.
- 다중 강의실 과목은 사용하는 강의실 수만큼 패널티가 누적된다.
- 최적값 허용 폭: `0점`. 즉, 최적 패널티보다 큰 해는 허용하지 않는다.
- 화면 점수: `max(0, 1 - 불리한 배정 수 / 20)`.
- 점수 예: 불리한 배정 0개는 `1.0점`, 4개는 `0.8점`, 20개 이상은 `0점`이다.
- 가중치와 지수는 각각 `1`, `1`이다.

### SC-02. 교수 수업 요일 수 3일 이하 선호

한 교수가 여러 과목을 맡을 때, 수업 요일 수를 3일 이하로 줄인다.

#### 상세 설정값·수치

- 대상 교수: 고정 시간이 아닌 과목을 `2개 이상` 맡은 주 담당 교수.
- 팀티칭 추가 교수는 계산에서 제외한다.
- `IsFixed=true` 과목은 계산에서 제외한다.
- 기준 요일 수: `3일`.
- 교수 1명의 패널티: `max(0, 사용 요일 수 - 3)`.
- 5일제이므로 교수 1명당 가능한 최대 초과값은 `2일`이다.
- 예: 3일 사용은 패널티 0점, 4일은 1점, 5일은 2점이다.
- 최적값 허용 폭: 총 초과 일수 `1일`.
- 화면 점수: `1 - 총 초과 일수 / (대상 교수 수 × 2)`.
- 가중치와 지수는 각각 `1`, `1`이다.

### SC-03. 같은 과목 블록의 요일 간격 2일 선호

블록이 여러 개인 과목은 블록들의 요일 차이가 2일이 되도록 선호한다.

#### 상세 설정값·수치

- 대상: `BlockStructure`에 블록이 `2개 이상`인 과목.
- 솔버 패널티는 `IsFixed=false`라서 블록 요일 변수가 생성된 과목에만 적용한다.
- 화면 점수는 `BlockStructure`가 2개 이상인 고정 시간 과목도 포함해 계산한다.
- 블록 쌍마다 요일 차이의 절댓값을 계산한다.
- 솔버 패널티: 차이 2일은 0점, 1일은 1점, 0·3·4·5일은 4점이다.
- HC-20 때문에 자동 배정 블록의 요일 차이 0일은 허용되지 않는다.
- 최적값 허용 폭: `0점`.
- 화면 점수: 차이 2일은 `1.0점`, 1일은 `0.8점`, 3일 또는 4일은 `0.2점`, 그 외는 `0점`이다.
- 블록이 3개 이상이면 모든 블록 쌍의 점수를 평균 낸다.
- 가중치와 지수는 각각 `1`, `1`이다.
